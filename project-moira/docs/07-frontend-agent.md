# 07 — Frontend Agent

> If this project doesn't need this agent, mark this doc `[NOT APPLICABLE TO THIS PROJECT]` at the top and leave the generic process rules below intact for future reference.

## Role
Implements all user-facing UI and client-side logic for Moira.

## Responsibilities
- Build React components, pages, and hooks per the architecture defined in `02-architecture.md`
- Implement card data model, spread definitions, and interpretation content as static TypeScript modules
- Build card animations (flip, deal, shuffle) using CSS transforms and React state
- Implement PNG/PDF export via `html2canvas` and `jsPDF`
- Ensure full responsive behavior from 320px to 1440px+
- Write tests per `18-testing-standard.md`
- Follow `17-coding-standard.md` for all code produced

## Inputs this agent expects
- Design direction and visual system (colors, typography, spacing, motion)
- Card interpretation content (can be generated by sub-agents in parallel)
- Spread definitions (names, positions, layouts)
- Card artwork assets (Rider-Waite-Smith public domain images)

## Outputs this agent produces
- Working React application (built with Vite)
- Static card and spread data modules
- Service layer (cardService, spreadService, exportService)
- Reading context and state management
- Animated card components
- Export functionality (PNG + PDF)
- Test files for services and components

## Standards it must follow
- `17-coding-standard.md` (TypeScript, React patterns, Tailwind usage)
- `18-testing-standard.md` (unit tests for services, component tests)
- `00-project-rules.md` (always)

## Known constraints / gotchas for this project
- No backend — all data is static; no fetch/axios calls at runtime
- `html2canvas` can be finicky with CSS transforms and shadows — test export early with complex layouts
- 78 card images must be lazy-loaded; preload only the card back and common cards
- Mobile: 10-card Celtic Cross layout is tight — use scrollable spread board on small screens
- Card flip animation uses 3D CSS transforms — test on Safari (known issues with `backface-visibility`)
